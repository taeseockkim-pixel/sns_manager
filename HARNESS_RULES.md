# SNS 관리 에이전트 하네스 규칙

> CIMON SNS 에이전트의 모든 워크플로우 스크립트는 이 규칙을 준수해야 합니다.

## 필수 원칙 (절대 규칙 — 위반 즉시 수정)

### H-01: Human-in-the-Loop 절대 원칙
```
모든 게시물은 사람의 명시적 승인(approved: true) 없이 자동 게시 불가.
publish 단계 전 반드시 approved: true 확인.
```
**이유**: IPO 준비 기업의 잘못된 자동 게시는 법적·홍보 리스크

### H-02: StructuredOutput 강제
```js
// 콘텐츠 생성 에이전트는 반드시 schema 옵션 지정
agent("...", { schema: CONTENT_SCHEMA })
```
**이유**: 파싱 오류로 인한 비정상 콘텐츠 게시 방지

### H-03: 브랜드 안전 검사 필수
```
brand_safety_score < 80 → 자동 반려 (사람 검토 불필요)
adversarial 에이전트 3개가 법적·톤·IPO 관점 독립 검토
```
**이유**: IPO 민감 정보 노출, 법적 리스크, 브랜드 훼손 방지

### H-04: 에러 에스컬레이션
```
API 호출 실패 → 재시도 없이 즉시 담당자 알림
자동 복구 금지
```
**이유**: 오류 은폐 방지, 빠른 인간 개입 보장

### H-05: 플랫폼 격리
```
X, Meta(FB/IG), LinkedIn 에이전트는 독립 실행
크로스 플랫폼 의존성 금지
```
**이유**: 단일 플랫폼 장애가 전체 시스템에 영향 주지 않도록

---

## 구현 규칙

| 규칙 | 내용 | 이유 |
|------|------|------|
| **H-06** | 워크플로우 스크립트 상단에 `export const meta = {...}` 필수 | 진행상황 추적 |
| **H-07** | 영문/국문 콘텐츠는 `parallel()` 동시 생성 | 지연 최소화 |
| **H-08** | 모든 단계에서 `log()` 호출 | 가시성 확보 |
| **H-09** | 최대 동시 에이전트 8개 이하 | API rate limit |
| **H-10** | worktree isolation 사용 안 함 | API 호출 중심 |
| **H-11** | 콘텐츠 캘린더 기반 스케줄링만 허용 | 임의 게시 방지 |
| **H-12** | 승인 대기 콘텐츠는 SQLite approval_queue에 저장 | 감사 추적 |

---

## 워크플로우 파이프라인

```
주제 입력
  → 03_generate_content.js  (영/국문 병렬 생성, H-07)
  → 04_brand_safety.js      (adversarial 검사, H-03)
  → 05_approval_gate.js     (담당자 승인 대기, H-01)
  → 06_publish.js           (승인 확인 후 게시, H-01 재확인)
```

**중요**: 04에서 score < 80이면 05, 06 실행 불가.
**중요**: 05에서 approved = false이면 06 실행 불가.
